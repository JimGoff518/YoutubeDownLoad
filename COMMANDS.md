# MARVIN Commands Cheat Sheet

## Session
| Command | What It Does |
|---------|--------------|
| `/marvin` | Start session — loads context, gives briefing |
| `/end` | End session — saves everything |
| `/update` | Quick checkpoint — saves progress without ending |
| `/wrapup` | Reviews session, updates ROADMAP + PROJECT_STATUS + DOCUMENTATION |
| `/handoff` | Generates copy-paste block for a new session |

## Work
| Command | What It Does |
|---------|--------------|
| `/commit` | Review changes and create git commits |
| `/report` | Generate a weekly summary |
| `/code` | Open MARVIN in your IDE |
| `/help` | Show commands and integrations |
| `/sync` | Pull updates from MARVIN template |

## Superpowers (used automatically)
| Skill | When It's Used |
|-------|---------------|
| `brainstorming` | Before building new features |
| `writing-plans` | Multi-step task planning |
| `executing-plans` | Running implementation plans |
| `systematic-debugging` | When bugs appear |
| `test-driven-development` | Before writing implementation code |
| `requesting-code-review` | After completing features |
| `verification-before-completion` | Before claiming work is done |
